# Entry Debugger 소개 웹페이지 디자이너 AI 전달 프롬프트

작성일: 2026-05-24  
대상 버전: Entry Debugger `2.0.0`  
참고 문서: `entry-debugger-supported-features.md`

아래 프롬프트는 디자이너 AI에게 그대로 전달하기 위한 문서입니다. 시각 디자인, 컬러, 타이포그래피, 레이아웃 스타일, 모션, 컴포넌트 미감은 전적으로 디자이너 AI가 판단하게 하고, 이 문서에서는 사이트에 반드시 들어가야 할 정보 구조와 콘텐츠 요소만 구체적으로 지정합니다.

---

## 디자이너 AI용 프롬프트

너는 Chrome 확장 프로그램 소개 페이지를 설계하는 디자이너 AI다.  
아래 정보를 바탕으로 `Entry Debugger`라는 확장 프로그램의 소개 웹페이지를 만들어라.

디자인의 시각적 방향, 레이아웃, 색상, 타이포그래피, 아이콘 스타일, 이미지 처리 방식, 섹션 간 리듬, 반응형 구성은 모두 너의 전문 판단에 맡긴다. 단, 아래에 명시한 정보 구조, 필수 콘텐츠, 기능 설명, 신뢰/보안 메시지, 캡처 이미지 배치 후보는 빠뜨리지 말아라.

### 1. 제품 개요

제품명: `Entry Debugger` 또는 `엔트리 디버깅 툴`

한 줄 설명:

> 엔트리 만들기 화면에서 변수, 리스트, 신호, 장면, 함수 내부 사용 위치, 콘솔 로그를 더 빠르게 확인하고 조작할 수 있게 해주는 디버깅 보조 확장 프로그램.

대상 사용자:

- 엔트리로 복잡한 작품을 만드는 사용자
- 변수, 리스트, 신호, 장면이 많아져 디버깅이 어려운 사용자
- 함수 내부에서 어떤 변수나 신호가 쓰였는지 추적해야 하는 사용자
- 엔트리 작품의 실행 흐름을 빠르게 검사하고 싶은 사용자
- 수업, 동아리, 개인 프로젝트에서 엔트리 작품을 자주 테스트하는 사용자

핵심 가치:

- 엔트리 화면 밖으로 나가지 않고 상태를 확인한다.
- 변수와 리스트 값을 빠르게 고친다.
- 함수 내부 사용 위치를 바로 찾아간다.
- 말하기 블록을 콘솔 로그처럼 활용한다.
- 실험실 기능으로 고급 디버깅과 편의 기능을 선택적으로 켠다.

### 2. 반드시 들어가야 할 최상단 영역

페이지 첫 화면에는 다음 요소를 배치한다.

- 제품명: `Entry Debugger`
- 한국어 제품명 또는 보조명: `엔트리 디버깅 툴`
- 메인 헤드라인 후보:
  - `엔트리 디버깅을 더 빠르게`
  - `변수, 리스트, 신호, 함수 사용 위치를 한 화면에서`
  - `엔트리 작품 테스트를 위한 디버깅 도구`
- 보조 설명:
  - `엔트리 만들기 화면에 디버깅 탭을 추가해 변수와 리스트를 실시간으로 보고 수정하고, 신호와 장면을 바로 테스트하며, 함수 안에서 사용된 자료의 위치까지 찾아줍니다.`
- CTA 버튼 후보:
  - `Chrome 웹스토어에서 설치`
  - `기능 살펴보기`
- 보조 정보:
  - `지원 URL: https://playentry.org/ws/*`
  - `권한: storage`
  - `Manifest V3`
- 첫 화면에 배치할 대표 이미지:
  - 엔트리 만들기 화면 오른쪽에 `디버깅` 탭이 열린 모습
  - 또는 확장 팝업과 디버깅 탭을 함께 보여주는 합성/프레임 이미지

### 3. 페이지 전체 권장 섹션

다음 섹션들이 자연스럽게 이어지도록 설계한다. 섹션 제목과 순서는 더 좋은 흐름이 있다면 조정해도 되지만, 내용은 모두 포함해야 한다.

#### 3.1 문제 제기 섹션

사용자가 겪는 문제를 짧고 명확하게 보여준다.

포함할 문제:

- 변수와 리스트가 많아지면 현재 값을 확인하기 어렵다.
- 신호가 제대로 발생했는지 테스트하기 번거롭다.
- 장면을 바꿔가며 실행 흐름을 확인하는 과정이 느리다.
- 엔트리 기본 속성 탭은 함수 내부 사용 위치를 충분히 자세히 보여주지 않는다.
- 함수 안에서는 개인 변수/리스트를 선택하기 어려운 경우가 있다.

#### 3.2 핵심 기능 요약 섹션

카드, 리스트, 탭, 스텝 등 어떤 형태든 좋다. 다음 핵심 기능 4개를 한눈에 보여준다.

1. `디버깅 탭`
   - 변수, 리스트, 신호, 장면을 한 패널에서 확인
   - 값 수정과 테스트 조작 지원

2. `함수 사용 바로가기`
   - 속성 탭에 `함수에서 사용` 섹션 추가
   - 어떤 함수의 어떤 블록에서 쓰였는지 찾아감

3. `콘솔 디버깅`
   - 말하기 블록에 `외치기`, `[LOG]`, `[INFO]`, `[WARN]`, `[ERROR]`, `[DEBUG]` 옵션 추가
   - 작품 실행 중 메시지를 콘솔 로그처럼 확인

4. `함수 안에서 개인변수 보기`
   - 함수 편집 중에도 현재 오브젝트의 개인 변수/리스트 선택 보조
   - 프로젝트 JSON을 직접 변환하지 않고 드롭다운 생성만 보조

#### 3.3 디버깅 탭 상세 섹션

디버깅 탭 안의 하위 탭을 소개한다.

포함할 하위 탭:

- `변수`
  - 현재 값 표시
  - 값 직접 수정
  - 검색
  - 스코프 표시 및 변경: `일반`, `공유`, `실시간`, `지역: 오브젝트이름`

- `리스트`
  - 항목 펼치기/접기
  - 항목 수정
  - 항목 추가
  - 항목 삭제
  - 스코프 표시 및 변경

- `신호`
  - 등록된 신호 목록 표시
  - 신호 검색
  - `신호 보내기`로 즉시 발생

- `장면`
  - 장면 목록 표시
  - 장면 검색
  - 선택 장면으로 이동
  - 실행 중 장면 전환 테스트 보조

이 섹션에는 실제 디버깅 탭 화면 캡처를 크게 보여주는 구성이 적합하다.

#### 3.4 함수 사용 바로가기 상세 섹션

이 기능은 사용자의 불편을 명확히 비교해서 보여준다.

반드시 포함할 설명:

- 엔트리 기본 기능은 함수 내부에서 쓰인 자료를 `함수에 조립되어 있어요.`처럼 간단히 표시한다.
- Entry Debugger는 기존 표시 아래에 `함수에서 사용` 목록을 추가한다.
- 목록에는 함수 이름과 사용된 블록 정보가 표시된다.
- 클릭하면 해당 함수 편집 화면으로 이동하고, 가능한 경우 해당 블록을 선택한다.

지원 대상:

- 변수
- 리스트
- 신호
- 함수

크레딧:

- `아이디어 제공: kkomaweb.com`

#### 3.5 콘솔 디버깅 상세 섹션

말하기 블록을 디버깅 로그처럼 사용할 수 있다는 점을 강조한다.

포함할 내용:

- 대상 블록: `말하기`, `N초간 말하기`
- 추가 옵션:
  - `외치기`
  - `[LOG]`
  - `[INFO]`
  - `[WARN]`
  - `[ERROR]`
  - `[DEBUG]`
- `외치기`는 엔트리에 이미 구현된 별 모양 말풍선을 사용한다.
- 로그 옵션은 말풍선을 띄우는 대신 콘솔에 메시지를 출력한다.
- 기존 프로젝트 JSON을 자동 변환하지 않는다.

크레딧:

- `아이디어 제공: aqu3180.co.kr`

#### 3.6 실험실 섹션

실험실은 기본적으로 꺼져 있고, 사용자가 필요할 때 켜는 고급 기능 모음으로 소개한다.

포함할 기능:

- `초시계/대답 디버깅`
  - 엔트리 기본 제공 변수인 `초시계`, `대답`의 현재 값 표시
  - 값 수정
  - 보이기/숨기기
  - 숨기기는 좌표 `(500, 0)`으로 이동
  - 보이기는 좌표 `(0, 0)`으로 이동

- `터보 모드`
  - 속도 조절 패널에 `∞` 단계 추가
  - `1`, `15`, `30`, `45`, `60`, `∞` 숫자 표시
  - `∞` 선택 시 FPS는 60으로 유지하고 `Entry.isTurbo` 활성화
  - 켜질 때 속도 버튼이 깜빡여 상태 변화 표시

- `속성 검색으로 찾기`
  - 변수/리스트/신호 드롭다운에 검색 UI 추가
  - 키워드 검색
  - 방향키 이동
  - Enter 선택
  - Escape 닫기

- `다량 이미지 업로더`
  - 여러 이미지 선택
  - 드래그 앤 드롭
  - 오브젝트 이름 지정
  - `.eo 다운로드`
  - 다운로드한 `.eo`를 엔트리의 `오브젝트 추가하기 > 파일 업로드`에서 업로드하라는 안내
  - 10MB 초과 시 업로드 실패 가능성 경고
  - BMP 제외
  - PNG/JPG/GIF/WEBP는 PNG로 변환
  - SVG는 SVG 원본과 PNG 미리보기/썸네일 생성

#### 3.7 부스트 모드 섹션

부스트 모드는 디버깅 탭 내부 설정과 엔진 화면 상단 버튼으로 나뉘어 동작하는 별도 기능으로 소개한다.

포함할 설명:

- 엔트리 WebGL 렌더링 옵션을 켜는 기능
- 내부적으로 `Entry.options.useWebGL`을 활성화
- 새로고침 후 반영
- 디버깅 탭의 `설정` 화면에서 `부스트 모드 버튼` 토글은 엔진 화면 상단 버튼 표시 여부를 제어
- 실제 부스트 ON/OFF는 엔진 화면 상단 번개 버튼에서 제어
- 기본값은 버튼 표시 켜짐, 실제 부스트 꺼짐
- 무거운 그래픽 프로젝트에서 렌더링 성능 개선을 기대할 수 있음

#### 3.8 안전한 동작 범위 / 신뢰 섹션

Chrome 웹스토어 등록용 소개 페이지이므로 신뢰 메시지를 분명하게 보여준다.

반드시 포함할 내용:

- 확장은 `https://playentry.org/ws/*`에서만 동작한다.
- 필요한 권한은 `storage`다.
- Manifest V3 기반이다.
- 확장 설정은 브라우저 저장소에 저장된다.
- 실험실 기능은 기본적으로 꺼져 있다.
- 값 수정, 리스트 편집, 스코프 변경, 오브젝트 추가는 사용자가 직접 버튼을 누른 경우에만 현재 프로젝트 상태에 반영된다.
- 콘솔 디버깅, 속성 검색, 함수 안 개인변수 보기는 프로젝트 JSON을 자동 변환하지 않는다.
- 이 확장 프로그램은 엔트리 공식 서비스가 아닌 개인 개발 도구다.
- 엔트리 내부 구조를 활용하는 일부 기능은 엔트리 업데이트에 따라 동작이 달라질 수 있다.

#### 3.9 설정 탭 섹션

디버깅 탭 내부 설정 화면에서 기능을 개별로 켜고 끌 수 있음을 보여준다. 확장 팝업은 디버깅 탭 표시 토글만 제공한다.

설정 탭에 있는 토글:

- 함수 사용 바로가기
- 콘솔 디버깅
- 함수 안에서 개인변수 보기
- 부스트 모드 버튼
- 실험실 탭

팝업에 표시되는 안내:

- `새로고침 해야 반영됩니다.`

#### 3.11 FAQ 섹션

다음 질문과 답변을 포함한다.

Q. 어디에서 동작하나요?  
A. `https://playentry.org/ws/*` 형태의 엔트리 만들기 화면에서 동작합니다.

Q. 어떤 권한을 사용하나요?  
A. 기능별 켜기/끄기 설정을 저장하기 위해 `storage` 권한을 사용합니다.

Q. 부스트 모드는 바로 적용되나요?  
A. 아니요. WebGL 렌더링 옵션은 Entry 초기화 시점에 반영되므로 새로고침이 필요합니다.

Q. 실험실 기능은 기본으로 켜져 있나요?  
A. 아닙니다. 실험실 탭과 실험 기능은 기본적으로 꺼져 있으며 사용자가 직접 켤 수 있습니다.

Q. 콘솔 디버깅은 프로젝트 JSON을 바꾸나요?  
A. 확장이 기존 프로젝트 JSON을 자동 변환하지 않습니다. 로그 옵션은 실행 중 콘솔 출력 보조 기능입니다.

Q. 함수 안 개인변수 보기로 만든 블록은 확장 없이도 실행되나요?  
A. 블록은 변수/리스트 ID를 참조하므로 실행 자체는 가능합니다. 다만 선택 UI 보조 기능은 확장이 있을 때 제공됩니다.

### 4. 페이지에 들어갈 주요 텍스트 후보

디자이너는 아래 문구를 그대로 쓰거나 더 자연스럽게 다듬어도 된다.

Hero 문구:

> 엔트리 디버깅을 더 빠르게

보조 문구:

> 변수와 리스트를 실시간으로 보고 수정하고, 신호와 장면을 바로 테스트하며, 함수 안에서 사용된 자료의 위치까지 찾아보세요.

핵심 기능 요약:

> 엔트리 화면 안에 디버깅 패널을 더합니다. 프로젝트를 왔다 갔다 하지 않고 현재 상태를 확인하고 필요한 테스트를 바로 실행할 수 있습니다.

안전 범위 문구:

> Entry Debugger는 `https://playentry.org/ws/*` 만들기 화면에서만 실행되며, 설정 저장을 위한 `storage` 권한만 사용합니다.

실험실 문구:

> 고급 기능은 실험실에 모았습니다. 필요할 때만 켜고, 프로젝트에 맞게 선택해서 사용하세요.

### 5. 디자이너 AI의 산출물 요구

다음 중 가능한 형태로 결과를 제안하라.

- 소개 페이지 전체 정보 구조
- 각 섹션의 배치와 콘텐츠 우선순위
- 페이지에 들어갈 실제 한국어 문구
- 각 섹션에 들어갈 이미지 위치와 설명
- CTA 배치
- 데스크톱/모바일 반응형 구성 방향
- 필요한 추가 캡처 이미지 목록

시각 디자인은 너의 전문 판단에 맡긴다. 단, 이 제품은 실제 작업 도구이므로 사용자가 기능을 빠르게 이해하고 신뢰할 수 있어야 한다.

## 소개 페이지에 필요한 캡처 이미지 리스트

아래 이미지는 디자이너 AI 또는 페이지 제작자가 준비하면 좋다. 모든 이미지를 반드시 넣을 필요는 없지만, `필수` 이미지는 소개 페이지의 설득력에 큰 영향을 준다.

### 필수 캡처

1. `확장 팝업 전체`
   - 디버깅 탭 토글만 보이는 화면
   - 현재 `manifest.json` 버전이 보이면 좋음
   - 용도: 팝업 진입점, 신뢰 섹션

2. `디버깅 탭 - 변수`
   - 엔트리 속성 패널에 `디버깅` 탭이 추가된 모습
   - 변수 목록, 값 표시, 스코프 선택이 보이는 화면
   - 용도: Hero 또는 디버깅 탭 상세 섹션

3. `디버깅 탭 - 리스트`
   - 리스트가 펼쳐져 있고 항목 수정/추가/삭제 UI가 보이는 화면
   - 용도: 리스트 기능 설명

4. `함수 사용 바로가기`
   - 엔트리 속성 탭의 기존 사용 위치 표시 아래에 `함수에서 사용` 목록이 추가된 화면
   - 용도: 함수 사용 바로가기 상세 섹션

5. `콘솔 디버깅 옵션`
   - 말하기 블록 드롭다운에 `외치기`, `[LOG]`, `[INFO]`, `[WARN]`, `[ERROR]`, `[DEBUG]`가 보이는 화면
   - 용도: 콘솔 디버깅 상세 섹션

### 권장 캡처

6. `실험실 탭`
   - 터보 모드, 다량 이미지 업로더, 속성 검색으로 찾기 토글
   - 초시계/대답 카드가 보이는 화면
   - 용도: 실험실 섹션

7. `터보 모드 속도 패널`
   - 속도 조절 패널에 `1`, `15`, `30`, `45`, `60`, `∞`가 표시된 화면
   - 가능하면 `∞`가 선택된 상태
   - 용도: 터보 모드 설명

8. `속성 검색으로 찾기`
   - 변수/리스트/신호 드롭다운을 열었을 때 검색 입력창과 필터링된 결과가 보이는 화면
   - 용도: 속성 검색 기능 설명

9. `다량 이미지 업로더`
   - 업로더 탭에서 파일 목록, `.eo 다운로드`, 파일 업로드 안내, 10MB 경고 상태가 보이는 화면
   - 용도: 업로더 기능 설명

10. `함수 안에서 개인변수 보기`
    - 함수 편집 중 변수/리스트 드롭다운에서 개인 변수 또는 개인 리스트가 보이는 화면
    - 용도: 함수 안 개인변수 보기 설명

### 선택 캡처

11. `신호 탭`
    - 신호 목록과 `신호 보내기` 버튼이 보이는 화면
    - 용도: 디버깅 탭 하위 기능 보강

12. `장면 탭`
    - 장면 목록과 `이동` 버튼이 보이는 화면
    - 용도: 장면 테스트 기능 보강

13. `E.DEBUG 콘솔 출력`
    - 브라우저 콘솔에 `[E.DEBUG]` 값 변경 로그가 찍힌 화면
    - 용도: 고급 디버깅 기능 보강

14. `부스트 모드 버튼`
    - 디버깅 탭 설정의 `부스트 모드 버튼` 표시 토글과 엔진 화면 상단 번개 버튼
    - 용도: 부스트 모드 설명

## 캡처 준비 메모

- 캡처는 실제 엔트리 만들기 화면 기준으로 준비한다.
- 개인정보, 프로젝트 ID, 사용자 계정 정보가 보이지 않도록 정리한다.
- 기능별 화면은 너무 많은 정보를 담기보다 해당 기능이 한눈에 보이게 찍는다.
- Chrome 웹스토어용 이미지와 웹페이지용 이미지는 비율이 다를 수 있으므로 원본 캡처는 넉넉한 해상도로 보관한다.
- 디자이너가 필요하다면 필수 캡처를 기반으로 확대 컷, 주석 컷, 합성 목업을 만들 수 있다.
